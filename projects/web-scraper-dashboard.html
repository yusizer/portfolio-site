Web Scraper + Dashboard

Enter a URL — the scraper pulls out price-shaped snippets (or exactly the elements you select) and shows them in a dashboard with persistent history. A JSON API is included for piping data elsewhere.

01Context

Businesses that sell or compete on price want fresh competitor pricing in one place instead of opening ten tabs a day. The job here was a reusable scraper that works on any page out of the box, plus a precise mode for known site structures.

02Problem

Manual copy-paste is slow, error-prone, and doesn't scale. Generic scrapers break on every different layout. The buyer wants either "just give me the prices on this page" (no setup) or "extract exactly these elements" (precise) — and a record of what was scraped.

03Solution

  • Generic mode: scans any page for price-shaped snippets ($12.99, €19.95, 250 000 сум, 1 299 руб, USD 50…) with surrounding context — no selector needed.
  • Precise mode: pass a CSS selector (e.g. .product-card) to extract exactly those elements — reliable for a known structure.
  • Dashboard: web UI with a form, results table, and persistent history of every scrape (SQLite).
  • JSON API: GET /api/scrape?url=…&selector=… for piping into sheets, dashboards, or alerts.
  • Robust: browser-like User-Agent, timeouts, graceful error handling (bad URL → clean error, not a crash).
  • Tests: 12 cases — price extraction, selector/generic parsing, API routes, form redirect, history.

04Outcome

Fresh competitor pricing on demand — type a URL, get a table — plus a JSON API to feed it into whatever the business already uses.

Why this matters (third-party benchmarks, cited)

  • Businesses using automated price monitoring react to competitor changes in hours, not days/weeks.
  • Lead-capture and data-pipeline automation is among the most-requested freelance tasks in 2026 (see botract, aziqdev).

05Tradeoffs & what I'd improve

  • Static HTML only — many sites render prices via JavaScript; for those the next step is a headless browser (Playwright) to scrape the rendered DOM.
  • Add scheduling + alerts — run daily and email/Telegram a diff when a price drops below a threshold.
  • Respect robots.txt & rate-limit for production use against third-party sites.
