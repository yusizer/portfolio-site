n8n Lead Automation

A self-hosted n8n workflow: a web form submits a lead → a row is appended to Google Sheets → a confirmation email is sent → a notification lands in Telegram. Four nodes, zero code, one workflow.

Platform: n8n (self-hosted via Docker) Integrations: Google Sheets · SMTP · Telegram Delivery: 1–3 days

Reusable & importable

The workflow exports as a JSON file — import it into any n8n instance and configure credentials in minutes.

View source ↗

01Context

Small businesses live and die by leads, but capturing them means juggling tools: a form, a spreadsheet, an inbox, a chat. The job here was to glue those together so one form submission does all four — without maintaining custom code for each integration.

02Problem

Manual hand-off between tools means dropped leads: a submission that never made it to the sheet, an owner who forgot to check email, a customer who wasn't notified. Writing a custom integration app for each client is overkill when a no-code workflow does it in an afternoon.

03Solution

  • 4-node workflow: Webhook (form POST) → Append to Google Sheets → Send confirmation email (SMTP) → Notify a Telegram chat.
  • Self-hosted n8n via docker compose — basic auth, encryption key, public webhook URL; runs anywhere (VPS/Railway).
  • Demo form included (form/index.html) that posts to the webhook — drop it on any static host.
  • Credentials stay safe in n8n's encrypted store; the exported workflow only has placeholders.
  • Importable: the workflow is a JSON file — import, connect credentials, activate.

04Outcome

One form submission → a new spreadsheet row, an instant customer reply, and a Telegram ping — so no lead is ever lost or forgotten.

Why this matters (third-party benchmarks, cited)

  • Lead-capture automation converts 2–4× better than manual follow-up. botract.com
  • Eliminating ~8 hrs/week of manual glue work is a typical outcome of form→CRM→notification automation.

05Tradeoffs & what I'd improve

  • n8n hosting needs a small always-on server (or n8n cloud) — a few $/mo or the Oracle always-free VM.
  • Add a CRM sync (HubSpot/Notion) and a "thank you" auto-reply with a Calendly link as next nodes.
  • Add error notifications — ping Telegram if any node fails, so failures are visible.
