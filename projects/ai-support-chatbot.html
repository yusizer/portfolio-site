AI Support Chatbot (RAG)

A customer-support chatbot that answers from a knowledge base using TF-IDF retrieval — with optional GPT answers grounded in that context. Runs in demo mode with no API key, falls back gracefully, and hands off to a human when unsure.

Stack: FastAPI · pure-Python TF-IDF · OpenAI (optional) · SQLAlchemy 2 Delivery: 2–3 weeks Tests: 10 passing

01Context

Support teams answer the same handful of questions all day. An AI assistant that answers from the company's own knowledge base — without hallucinating — cuts repetitive load and replies instantly, 24/7.

02Problem

Generic chatbots hallucinate answers that sound confident but are wrong — dangerous for support. A pure LLM with no grounding invents policies, prices, and hours. The job was to ground every answer in the business's actual FAQ, and to know when to stop and hand off to a human.

03Solution

  • Retrieval (TF-IDF): pure-Python index over the FAQ knowledge base — scores each question by IDF-weighted overlap, returns the top matches. No heavy ML deps.
  • Two modes: local demo (no API key) returns the best-matching FAQ answer; LLM mode (with OPENAI_API_KEY) has GPT answer using the top-3 FAQs as context (RAG).
  • Grounding + guardrails: the system prompt forces answers to use only the provided info; if it's not covered, say so.
  • Graceful fallback: if the LLM call errors, the bot falls back to the retrieved FAQ answer — never a blank reply.
  • Human handoff: below a confidence threshold, the bot says it'll connect a human instead of guessing.
  • FAQ admin API + chat logs: add/manage knowledge-base entries; every conversation is logged for review.
  • Tests: 10 cases — retrieval, local/LLM/handoff modes, fallback on error, FAQ CRUD, logs.

04Outcome

Instant, grounded answers from the business's own knowledge base — with a demo that runs without any API key and a safe handoff when the bot isn't sure.

Industry context (third-party, cited)

  • Booking/support bots cut repetitive support load by up to 60%. chat2desk.com
  • RAG grounds LLM answers in source documents, the standard pattern to prevent hallucination in support bots.

05Tradeoffs & what I'd improve

  • TF-IDF vs embeddings: great for a small FAQ and fully offline; for a large/semantic KB I'd swap in vector embeddings (e.g. OpenAI embeddings + cosine).
  • No conversation memory yet — each turn is independent; next step is short-term memory for follow-ups.
  • Telegram channel: same backend behind an aiogram bot so answers reach customers where they already are.
